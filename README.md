# Верстак гипотез

Streamlit-интерфейс для генерации, ранжирования, экспертной проверки и экспорта научно-исследовательских гипотез.

## Реализовано

- KPI и технологические ограничения вводятся текстом;
- количество гипотез определяется генератором автоматически;
- механизм, новизна, риск, технологическая и экономическая ценность;
- ранжирование с изменяемыми весами трёх осей;
- учёт ограничений и промышленного масштаба;
- причинная цепочка и диаграмма влияния;
- оценка неопределённости и рекомендация по проверке;
- происхождение гипотезы с обязательными ссылками на источники;
- открытые источники разрешены и передаются backend через `use_open_sources=true`;
- эксперт может принять/отклонить гипотезу, отметить её проверенной, поставить оценку и оставить комментарий;
- текст гипотезы и системные оценки эксперт не редактирует;
- экспертные оценки сохраняются в SQLite;
- экспорт PDF, DOCX, CSV, JSON и CSV для Jira/YouTrack;
- русский, английский и китайский интерфейс;
- Docker и Docker Compose.

## Локальный запуск

```bash
py -3.14 -m pip install -r requirements.txt
py -3.14 -m streamlit run app.py
```

Для Python 3.12:

```bash
python -m pip install -r requirements.txt
streamlit run app.py
```

## Подключение backend

Задайте переменную окружения:

```text
HYPOTHESIS_API_URL=http://localhost:8000/generate
```

Запрос:

```json
{
  "kpi": "Повысить извлечение никеля из руды на 8%",
  "constraints": "Не увеличивать расход реагентов и использовать существующее оборудование",
  "language": "ru",
  "use_open_sources": true,
  "industrial_scale": true
}
```

Backend самостоятельно определяет количество релевантных гипотез.

Каждая гипотеза должна содержать:

```json
{
  "hypothesis": "...",
  "rationale": "...",
  "mechanism": "...",
  "expected_effect": "...",
  "industrial_scale": "...",
  "kpi_link": "...",
  "constraints_fit": "...",
  "novelty": {"score": 0.8, "why": "..."},
  "risk": {"score": 0.3, "why": "..."},
  "value": {"score": 0.9, "why": "..."},
  "economic_value": {"score": 0.8, "why": "..."},
  "success_probability": {"score": 0.3, "why": "..."},
  "verification_recommendation": "...",
  "causal_chain": ["воздействие", "механизм", "промежуточный эффект", "KPI"],
  "resource_estimate": {"time": "...", "cost": "...", "volume": "..."},
  "evidence": [
    {
      "triplet": "...",
      "source": "...",
      "source_url": "https://...",
      "page": 125,
      "chunk_id": "CH-001",
      "evidence_fragment": "...",
      "confidence": 0.87
    }
  ],
  "roadmap": [],
  "status": "draft"
}
```

`source_url` обязателен для каждого источника.

## Docker

```bash
docker compose up --build
```

Приложение откроется на `http://localhost:8501`.

## Хранение

Локально данные сохраняются в `data/workbench.db`. Для постоянного облачного хранения после перезапусков контейнера следует подключить Postgres или постоянный volume.


## Выбор базы знаний

В выпадающем поле можно выбрать:

- «Все базы знаний»;
- одну базу;
- несколько баз.

Поддерживаются научные публикации, патенты, внутренние отчёты, исторические эксперименты, данные о материалах, данные о процессах и открытые источники. В API передаётся массив `knowledge_bases`.

Показатель `success_probability` отображается в интерфейсе как «Вероятность успеха».
